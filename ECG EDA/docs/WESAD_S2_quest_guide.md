# WESAD `S2_quest.csv` 说明文档

这份文档用于帮助读者看懂：

- `WESAD/WESAD/S2/S2_quest.csv`

注意：

- 这是 **WESAD 原始数据文件**
- 当前项目**不会直接修改原文件**
- 这里只做解释和注释，保证原始数据可追溯

## 1. 这个文件为什么看起来不像普通 CSV

它虽然叫 `.csv`，但有两个特点：

1. 使用的是 `;` 作为分隔符，而不是逗号
2. 很多行以 `#` 开头，表示这是带标签的记录行

所以它更像一种“半结构化问卷元数据文件”，而不是规整的表格型 CSV。

## 2. 文件整体在记录什么

这个文件主要记录两类信息：

1. 受试者在实验中的各个阶段顺序与时间
2. 与实验阶段对应的问卷结果

也就是说，它是 **实验元数据 + 主观量表信息**，不是 ECG / EDA 原始波形本体。

## 3. 前几行是什么意思

例如：

```text
# Subj;S2
# ORDER;Base;TSST;Medi 1;Fun;Medi 2;sRead;fRead
# START;7.08;39.55;70.19;81.25;93.38;54.42;89.51
# END;26.32;50.3;77.1;87.47;100.15;56.07;91.15
```

可以理解为：

- `Subj`
  - 当前受试者是 `S2`
- `ORDER`
  - 实验阶段名称和顺序
- `START`
  - 每个阶段的开始时间
- `END`
  - 每个阶段的结束时间

### 阶段名称大致含义

- `Base`
  - 基线阶段
- `TSST`
  - 压力诱发阶段，通常是 Trier Social Stress Test
- `Medi 1 / Medi 2`
  - 冥想或恢复阶段
- `Fun`
  - 娱乐 / 正性情绪阶段
- `sRead / fRead`
  - 阅读类任务阶段

## 4. 后面的问卷块是什么

文件后面能看到多组问卷条目：

- `PANAS`
- `STAI`
- `DIM`
- `SSSQ`

这些不是生理信号，而是主观量表或情绪维度评分。

### `PANAS`

`PANAS` 通常表示积极与消极情绪量表。

在这个文件里会看到多行 `# PANAS;...`，表示一组条目评分。

### `STAI`

`STAI` 是状态-特质焦虑量表。

这部分对你的课题尤其有价值，因为它和“焦虑”概念更接近。

### `DIM`

通常可以理解为维度式的主观评分，例如唤醒度、效价等维度。

### `SSSQ`

通常与主观压力或认知负荷相关。

## 5. 这些问卷和生理信号是什么关系

你可以这样理解：

- ECG / EDA / 温度 / ACC 是 **客观生理信号**
- `S2_quest.csv` 里的内容是 **主观标签与实验元数据**

所以这个文件更适合作为：

- 阶段解释材料
- 问卷标签参考
- 后续标签增强或结果解释的辅助依据

而不是直接当作模型输入主表。

## 6. 为什么当前项目没有直接修改它

因为它属于原始数据文件，直接修改会带来几个问题：

1. 破坏数据原貌
2. 后续难以追溯
3. 不利于和公开 WESAD 原始版本保持一致

所以更稳妥的做法是：

- 保留原始文件
- 单独写说明文档
- 如有需要，再额外生成“清洗版说明表”

## 7. 对当前项目的实际作用

在你当前这个项目里，`S2_quest.csv` 的作用主要是：

1. 帮助解释 WESAD 每个实验阶段是什么
2. 帮助理解为什么某些窗口会被标成不同状态
3. 帮助你以后把主观问卷和生理信号做更正式的结合

## 8. 一句话总结

`S2_quest.csv` 不是原始波形数据，而是 WESAD 的问卷与实验阶段元数据文件。  
它更像“实验说明书 + 主观标签补充信息”，适合解释和辅助建模，不建议直接改原文件。
